Why traits?

Traits are a flexible mechanism to factor out and recombine reusable pieces of code. They a more robust alternative to multiple inheritance and mixins. They are more robust because name clashes must be resolved explicitly by composers, and because trait composition is order-independent (i.e. more declarative).

Why traits.js?

Because it's minimal, easy to understand and because it plays nicely with the new object-manipulation API defined on Object in Ecmascript 5th edition.

traits.js is compliant with any Javascript engine supporting Ecmascript 262 Edition 3 and Edition 5. This includes the engines of all major browsers. It does not depend on browser-specific additions.
